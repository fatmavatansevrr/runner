# Domain Blocker Source-Mapping Plan — SOURCE-MAP-001

Read-only. Maps each of the 13 active-root blockers (see `active-v4-domain-blocker-inventory.md`) to the
canonical source hierarchy defined in `docs/README.md` §1:

1. Approved Golden Fixture v3 (`docs/canonical/golden-fixture-v3/golden-10k-intermediate-4d-12w.v3.*`)
2. Approved versioned rule files referenced by Golden Fixture v3 (`progression_rules_v2.yaml`)
3. Approved Plan Generation Decisions documents (none currently exist under `docs/canonical/` — see note below)
4. Explicitly approved reconciled architecture specifications (`docs/specifications/`)
5. Plan Catalog canonical brief (`docs/specifications/plan-catalog-antigravity-brief-v2.md`)
6. Existing catalog artifacts and tests (lowest precedence; never sufficient alone)

**Note on tier 3**: no "Plan Generation Decisions" document currently exists under `docs/canonical/`. Any
future such document, once approved, would slot in at precedence level 3 — above the brief, below the
fixture/rule-files. Its absence is recorded here as a source-mapping fact, not assumed to exist.

Source silence is never treated as support or opposition (per `docs/README.md` §1: "If precedence does
not resolve the conflict, the decision remains `PLACEHOLDER_UNCONFIRMED`"). A generated fixture *instance*
value is never, by itself, sufficient to infer a general policy — every "fixture relevance" cell below
distinguishes a **directly stated fact** (evidence) from a **single realized instance** (not evidence of
a general rule, per `ten-k-pilot-domain-review-summary.md`'s explicit precedent for D2/D4).
